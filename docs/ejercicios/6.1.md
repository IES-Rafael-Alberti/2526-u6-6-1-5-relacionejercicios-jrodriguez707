II.c) ¿Existe otra forma de implementarlo que no sea herencia? (composición, interfaces, delegación, etc.)

RESPUESTA:
Por ejemplo, en lugar de heredar de `Persona`, se podría crear una interfaz llamada `Estudiable` con un método `estudiar()`.

Luego las clases que necesiten ese comportamiento podrían implementarla.

También se podría usar **composición**, donde una clase tenga otra clase como propiedad en lugar de heredar de ella.

II.d) ¿Qué ventajas aporta esta implementación con respecto a la alternativa?

RESPUESTA:

Primera propuesta:

Usar herencia permite representar de forma clara la relación **"es un tipo de"**.

Un `Coche` **es un tipo de** `Vehiculo`.

Ventajas:

- reutilización de código
- organización jerárquica clara
- posibilidad de usar polimorfismo



Segunda propuesta:

La herencia permite compartir comportamientos comunes, como el método `presentarse()` de `Persona`.

Además, cada subclase puede añadir nuevas funcionalidades específicas.

Ventajas:

- evita duplicar código
- facilita la extensión del sistema
- mejora la legibilidad del modelo de clases
